# TASK-050: fix: session resume broken on app restart — missing timestamp from DB load

## Description

ClaudeCodeProvider.loadSessionMap() only loads canvas_session_id and cli_session_id from canvas_cli_sessions table, but NOT updated_at. This means cliSessionLastUsed[id] = nil for all loaded sessions. getCliSession() guards on 'guard let lastUsed = cliSessionLastUsed[id]' and returns nil when timestamp is missing. Result: --resume is never passed to the CLI after any app restart, so every session starts fresh. Fix: load updated_at from DB and set cliSessionLastUsed[id] = updatedAt during loadSessionMap(). Sessions within 30-minute TTL will resume; beyond = fresh start.

- [ ] cliSessionLastUsed[id] is set to updatedAt timestamp during load

- [ ] --resume is passed to CLI after app restart for eligible sessions

## Context

**Why this matters:** Session continuity is broken after app restart. Every app launch loses context and starts fresh, defeating the purpose of session persistence.

**Root cause:** Missing timestamp data from DB load means TTL check always fails.
