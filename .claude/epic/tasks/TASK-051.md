# TASK-051: fix: canvas-sessions.json loses timestamps — add SessionEntry format with lastUsed

**Status:** Pending
**Priority:** High
**Assignee:** Unassigned
**Created:** 2026-03-02
**Updated:** 2026-03-02

## Description

canvas-sessions.json is stored as [String: String] (canvasId -> cliId) with no timestamp data. When loaded, no timestamps are set, compounding the DB timestamp bug. Fix: change file format to [String: SessionEntry] where SessionEntry has cliSessionId + lastUsed (unix timestamp). Use versioned decode with fallback to old format for migration.

---

## Acceptance Criteria

- [ ] SessionEntry struct created with cliSessionId and lastUsed fields
- [ ] canvas-sessions.json format updated to [String: SessionEntry]
- [ ] Decoder includes version field and fallback for old [String: String] format
- [ ] Old format sessions are migrated with current timestamp
- [ ] New sessions write lastUsed on every save
- [ ] Backward compatibility maintained through 2 app launches, then old format cleaned

---

## Context

**Why this matters:** File format lacks timestamp data, so sessions cannot be validated against TTL when app restarts.
