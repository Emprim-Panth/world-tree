## Description

FRD-005 | Real-time streaming view implemented with 60fps token batching.

## Implementation

All changes in `Sources/Features/Document/DocumentEditorView.swift`:

### 1. 60fps Token Batching
- `pendingTokenBuffer: String` — accumulates tokens between timer fires
- `streamFlushTimer: Timer?` — 60fps main-RunLoop timer (1/60s interval, `.common` mode)
- `startStreamBatching()` / `stopStreamBatching()` — lifecycle around the provider stream
- `flushPendingTokens()` — drains buffer into `streamingContent` once per frame
- Provider stream now writes to `pendingTokenBuffer` instead of directly to `streamingContent` — eliminates per-token SwiftUI re-renders

### 2. Auto-scroll with User-scroll Inhibit
- `isScrolledToBottom: Bool` — tracked via `onScrollGeometryChange` (distance-from-bottom < 80pt)
- `onChange(of: streamingContent)` only auto-scrolls when `isScrolledToBottom == true`
- `hasNewStreamContent` flag set when tokens arrive while scrolled up

### 3. "Scroll to Bottom" FAB
- `ScrollToBottomFAB` — animated arrow button with pulsing ring, appears bottom-trailing
- Shown when `hasNewStreamContent && !isScrolledToBottom`
- Snaps to streaming anchor or last section; clears the flag on tap
- `.transition(.scale.combined(with: .opacity))` + `.animation` for smooth show/hide

### 4. Pulsing Cursor
- `PulsingCursor` — replaces `BlinkingCursor`
- Scale + opacity + cyan glow animation (0.55s easeInOut repeating)
- `typealias BlinkingCursor = PulsingCursor` for zero-migration compatibility

### Acceptance Criteria
- [x] TokenStreamView renders tokens at 60fps without jank (batching prevents per-token re-renders)
- [x] Smooth character-by-character append animation (flushed in batches, not one char at a time)
- [x] Handles message_complete — `stopStreamBatching()` drains remaining tokens before `streamingContent = nil`
