## Description

User keeps getting repeated permission prompts despite approving them.

Root causes:
1. VoiceService.requestPermissions() calls request APIs unconditionally — needs status check first
2. NotificationManager.requestAuthorization() doesn't check existing auth status before requesting
3. NSOpenPanel in SidebarView.swift + ForkMenu.swift always resets to ~/Development (no memory)
4. CODE_SIGN_IDENTITY: "-" in project.yml = ad-hoc signing → TCC re-prompts after each binary rebuild

Fixes:
1. VoiceService: check AVCaptureDevice.authorizationStatus + SFSpeechRecognizer.authorizationStatus before calling request APIs
2. NotificationManager: check UNUserNotificationCenter.current().notificationSettings() status before requesting
3. NSOpenPanel: persist last-used directory in UserDefaults("lastWorkingDirectory")
4. project.yml: change CODE_SIGN_IDENTITY to use stable signing identity
