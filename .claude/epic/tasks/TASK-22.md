# TASK-22: Wire SessionEnd hook to POST summaries to ContextServer

**Status:** open
**Priority:** high
**Epic:** EPIC-WT-SIMPLIFY
**Phase:** 4 — Integration
**Depends on:** TASK-21

## Context

SessionEnd hook currently writes to various places (journal, compass, dispatch). This task adds a call to POST /session/summary to ContextServer so session summaries accumulate in BRAIN.md's "Recent Sessions" section — making knowledge persistent across compactions.

## Where to Edit

cortana-core HookProcessor, `SessionEnd` / `Stop` handler. Likely in:
`~/Development/cortana-core/src/hooks/session-end.ts`
(verify path before editing)

## Required Change

After existing session-end logic, add:

```typescript
async function postSessionSummary(
  project: string,
  summary: string,
  decisions: string[],
  corrections: string[]
): Promise<void> {
  const token = loadGatewayToken()

  try {
    await fetch('http://127.0.0.1:4863/session/summary', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-cortana-token': token
      },
      body: JSON.stringify({ project, summary, decisions, corrections }),
      signal: AbortSignal.timeout(3000)
    })
  } catch {
    // Fire and forget — session end should never fail due to ContextServer being down
  }
}
```

**Fire-and-forget with timeout.** Session end must complete even if World Tree is down. Never await or throw from this function.

## Summary Generation

The session summary should be extracted from the existing session state. Use whatever `summary` field is already generated by the SessionEnd hook. If no summary exists, use the last user message as a fallback.

## Decisions/Corrections Detection

The existing HookProcessor may already detect decisions and corrections (DecisionDetector, correction detection). Pass those through. If not, pass empty arrays — better to have partial data than to skip the call.
