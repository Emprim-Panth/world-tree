# TASK-052: fix: snapshot checkpoint only written on onDisappear — write after each response

**Status:** Done
**Priority:** High
**Assignee:** Unassigned
**Created:** 2026-03-02
**Updated:** 2026-03-02

## Description

writeSnapshotCheckpoint() is only called in .onDisappear. Force-quits and crashes bypass onDisappear, losing all context for short sessions that never hit 75% rotation pressure. Fix: also call writeSnapshotCheckpoint() after each assistant response completes (after stream .done event and section is persisted), so there's always a recent checkpoint even if the app crashes.

## Acceptance Criteria

- [ ] writeSnapshotCheckpoint() called after each assistant response stream completes

- [ ] No duplicate checkpoints for same response
- [ ] Performance impact minimal (write is already async)
